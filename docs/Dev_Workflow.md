# 개발 워크플로우

> 이 프로젝트에서 Claude Code와 협업하는 작업 흐름을 정의한다.

---

## 전체 흐름

```
KTX_Ticketing_Task_Checklist.md
        ↓
1. 다음 Phase/Task 추출
        ↓
2. 실행 계획 작성 (Plan mode)
        ↓
3. 사람 검토 및 승인
        ↓
4. 소단위 구현 반복
   ┌─────────────────────────────┐
   │  코드 작성                  │
   │      ↓                      │
   │  테스트 유효성 검토          │
   │      ↓                      │
   │  테스트 작성 및 실행         │
   │      ↓                      │
   │  커밋 메시지 초안 제시       │
   │      ↓                      │
   │  사람 승인 후 커밋           │
   └─────────────────────────────┘
        ↓
5. Phase 결과 보고서 작성 (docs/P{N}_Result.md)
        ↓
6. 체크리스트 완료 처리 + 커밋
        ↓
7. 다음 Phase로 반복
```

---

## 단계별 상세

### 1. Task 추출
- `KTX_Ticketing_Task_Checklist.md`에서 현재 Phase의 미완료(`[ ]`) 태스크 확인
- DoD(Definition of Done) 기준으로 완료 조건 파악

### 2. 실행 계획 작성
- Plan mode에서 구현 파일 목록, 커밋 단위, 검증 방법 작성
- 락된 아키텍처 결정(`CLAUDE.md`) 위반 여부 사전 점검

### 3. 사람 검토 및 승인
- 계획 승인 전까지 코드 작성 시작하지 않음
- 방향 수정은 이 단계에서

### 4. 소단위 구현 반복
- **테스트 유효성 검토**: 작성 전 "이 테스트가 비즈니스 로직/설계 결정을 검증하는가?" 확인
  - 무의미한 패턴: Java 언어 동작, H2 스키마, 단순 getter
- **커밋 단위**: 엔티티 추가 / 서비스 추가 / 테스트 추가 / 설정 변경 등 논리적 소단위
- **커밋 전**: 메시지 초안을 사람에게 제시 → 승인 후 실행

### 5. 결과 보고서
- `docs/P{N}_Result.md` 생성
- 내용: 검증 결과, 구현 파일 목록, 설계 결정 및 트레이드오프, 발견한 이슈, 커밋 이력

### 6. 체크리스트 업데이트
- 완료 태스크 `[ ]` → `[x]` 갱신
- 진행 현황 요약 테이블 업데이트

---

## 규칙 요약

| 규칙 | 내용 |
|------|------|
| 커밋 승인 | 커밋 전 메시지 초안 제시 → 사람 승인 후 실행 |
| 커밋 단위 | Phase 전체 묶음 금지. 논리적 소단위로 분리 |
| 테스트 | 작성 전 유효성 검토 필수. 무의미한 테스트 작성 금지 |
| 계획 | 승인 없이 구현 시작 금지 |
| 아키텍처 | CLAUDE.md의 락된 결정 사전 검토 필수 |
